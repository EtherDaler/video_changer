# video_changer — контекст для агента

## Назначение

Проект — **подмена или изменение объектов на видео по текстовому промпту**. Идея: не генерировать всю сцену, а найти объект, сгенерировать только его в ROI и склеить с исходным кадром.

## Пайплайн (целевой)

1. **Grounding DINO** — по промпту находит bbox объекта на кадре(ах).
2. **SAM2** — маска на seed-кадре, затем **video tracking** масок по всем кадрам.
3. **Gen-API Nano Banana 2** — inpaint **только ROI** (картинка ROI + маска ROI), без полнокадровой генерации. Опционально **третье изображение** — последний сгенерированный объект для режима **edit** (поворот/сдвиг при смене ракурса).
4. **Временная согласованность** (видео): по метрикам маски (центр, log площади, aspect) кадры с **малым** изменением переиспользуют один и тот же сгенерированный ROI (только resize под текущий bbox); при **среднем** изменении — запрос с референсом; при **сильном** — полная перегенерация. При пропадании объекта состояние сбрасывается.
5. **Склейка** с полным кадром: `refine_mask` → crop/resize ROI → ответ API → `alpha_insert_smooth` (опционально Poisson), как в single-frame ноутбуке. Для «хвостов» старого объекта и рамки вокруг вставки: `ROI_PADDING`, `MASK_DILATE`, затем **`MASK_COVER_EXTRA_DILATE`** / **`SMALL_MASK_AREA_*`** (ранние кадры с маленькой маской), **`BLEND_MASK_DILATE_PX`** и **`COMPOSITE_FEATHER_KS`** (мягкий край без прямоугольной кромки); при остаточных швах попробуй **`USE_POISSON_BLEND = True`**.

## Параметры и секреты

- Ключ **Gen-API**: переменная окружения `GEN_API_KEY` или явное присваивание в ячейке параметров **без коммита** реальных ключей в git.
- Промпты замены: шаблон вида `Replace {OLD} to a {NEW} ...` — в видео-ноутбуке обычно `OLD_OBJECT_PROMPT` / `NEW_OBJECT_PROMPT` (часто = `DETECT_PROMPT` и `PROMPT`).

## Ограничения

- Запросы к API только на **full** и **edit**; кадры с **reuse** (стабильная маска) идут без сети. Время и стоимость зависят от движения объекта и порогов `TEMP_*`.
- **Seed SAM2**: `SEED_MODE=earliest` — детекция с **самого раннего** среди найденных кадров (не только max confidence), иначе трекинг часто «живёт» только возле позднего seed. Увеличивай `SCAN_FRAMES`, если объект появляется раньше, чем попадает в выборку.
- **Экспорт**: кадры обрабатываются в `WIDTH×HEIGHT`, итог **апскейлится** под исходное разрешение; звук — через `ffmpeg` mux с `INPUT_VIDEO`.
- Не путать с устаревшим описанием «SDXL/Flux на весь кадр» — **текущая цель — ROI + Nano Banana 2**.

## Стиль изменений в коде

- Ноутбуки — сохранять работоспособность ячеек по порядку (Colab/локально).
